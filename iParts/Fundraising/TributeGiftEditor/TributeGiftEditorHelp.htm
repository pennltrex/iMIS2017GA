Tribute Gift Editor

  • Name - Enter a name for the content item. This text appears during design mode (configuration).
  • Title - Enter the text to display above this iPart on the rendered page. If blank, no title displays.
  • Do not render in design mode - Select this option to prevent content record data from being loaded in the content editor. This improves performance during design time (configuration).
  • iPart CSS class - (optional) Enter the name of a CSS class that will be applied to the iPart at runtime. Add this CSS class to an appropriate style sheet. Using such classes lets you apply special formatting to targeted iParts in a reusable way.
  • Display a border around this content - Select this option to display a border around this iPart on the rendered page.
  • Display content within a collapsible panel - Enable to allow users to minimize the panel within the page, to work with dense pages more efficiently.
    • Show the content as collapsed - Enable to display the panel minimized by default.
      Tip: Be sure that the Title makes clear what data is hidden from view.
  • Display on screen sizes (for responsive themes only):
    • Extra-small screens - When disabled, this content item is hidden on extra small devices, such as smartphones, where the width of the screen is less than 768px.
    • Small screens - When disabled, this content item is hidden on small devices, such as tablets, where the width of the screen is greater than 768px and less than 991px.
    • Medium screens - When disabled, this content item is hidden on medium devices, such as laptops, where the width of the screen is greater than 992px and less than 1199px.
    • Large screens - When disabled, this content item is hidden on large devices, such as wide computer monitors, where the width of the screen is greater than or equal to 1200px wide.
      Note: When Easy Edit is enabled, all content items for a page are displayed, regardless of the Display on screen sizes (for responsive themes only) options that are selected.
  • Display notification option - Select this option to show the "Please inform the following person about my donation" check box
  • Text for notification prompt - Enter the text you want to appear next to the checkbox users can select to notify someone of their donation. The default text is Please inform the following person about my donation.
  • Include email for individuals -  Deselect this option to hide the Email field.
    • Require email for individuals -  (only enabled if Include email is selected) Select this option to require the donor to include an email address for the person to be notified.
  • Include address -  Deselect this option to hide the Address fields.
    • Require address - Select this option to require the donor to enter an address for the person to be notified.
    • Multiple lines for street address - Select this option to display three Street address lines. If disabled, only a single line is displayed for the Street address.
    • Create address purpose - Select the address purpose to which the new address will be applied.
  • Commit sequence - Commit sequence affects how multiple iParts interact on a page. Enter a non-zero number to specify the order that changes are saved relative to other inline editable iParts on the rendered page. Zero (default) = save order is not important.
  • IQA query for predefined honorees - If needed, select a custom query for listing appropriate tribute names (honorees), to allow users to choose from a pre-defined list of contacts as they type a tribute name.
    • Note: If the query is not selected or the selected query is not found (such as when the query name is mistyped) the iPart will display a standard text box for the tribute name.
    • Note: Any query used must include key_UniformKey and FullName as Display Columns. Please see the default query (Memorial Tribute Honorees - Active at $/Fundraising/DefaultSystem/Queries/MemorialTribute/Memorial Tribute Honorees - Active) for an example.

Notes

  • The Tribute Gift Editor is an optional iPart that you can use in donation pages that contain the Donation Creator iPart configured to Process donations immediately. It cannot be connected to the Donation Creator when it is configured for Add donation to the cart mode.
  • A content record used for donation pages set up to process donations immediately must contain the following iParts with the Commit sequence value shown in parentheses:
    • Donation Creator (2)
    • (optional) Display Name Editor (3)
    • (optional) Tribute Gift Editor (4)
    • Contact Account Creator (1)
    • Payment Creator (5)
    • Submit Order Button Link (6)
  • To use this iPart in a donation page, you must create a connection between it and the Donation Creator iPart. To set up the connection: From the Tribute Gift Editor iPart's title-bar menu, select Connect. Select Create new connection . From the Object Consumer dropdown list, select your Donation Creator iPart. Select Connect, then Save and Publish the content record. See Connecting iParts to refresh panels

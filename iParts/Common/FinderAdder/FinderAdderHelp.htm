Finder Adder

Under development for future release

The Finder/Adder iPart is primarly used in system setup pages to display specific queries for finding or adding people or organizations. The Finder/Adder is not intended to be added as a standalone iPart as it is used with other iParts.

  • Name - Enter a name for the content item. This text appears during design mode (configuration).
  • Title - Enter the text to display above this iPart on the rendered page. If blank, no title displays.
  • Do not render in design mode - Select this option to prevent content record data from being loaded in the content editor. This improves performance during design time (configuration).
  • iPart CSS class - (optional) Enter the name of a CSS class to associate with this iPart, which will add a DIV so named around the iPart. Add this CSS class to an appropriate style sheet. Using such classes lets you apply special formatting to targeted iParts in a reusable way.
  • Display a border around this content - Select this option to display a border around this iPart on the rendered page.
  • Display content within a collapsible panel - Enable to allow users to minimize the panel within the page, to work with dense pages more efficiently.
    • Show the content as collapsed - Enable to display the panel minimized by default.
      Tip: Be sure that the Title makes clear what data is hidden from view.
  • Document path - Select the path in the Document System to the query or folder of queries to use to find the desired information.
  • Business object name - The name of the business object represented in the grid. Required if Allow Deleting is checked.
  • Properties to display - A comma separated list of properties in the query or business object to display. If empty, all properties will be displayed.
  • Sort by properties - A comma separated list of properties in the query or business object to sort by. If empty, the grid will sort in default order.
  • Target Content Code - If selecting an item in the list, allows for navigation to a page containing the unique content code entered in this field.
  • Target URL - If selecting an item in the list, allows for navigation to a page containing the unique URL link entered in this field.
  • Open links in the same window - If selected, linked pages (select and add) will open within the existing navigation instead of a pop-up.
  • Allow adding - Enable to allow users to add items to the grid.
    • Add content code - If users can add new items, this defines the page containing the content code of the add page. If not defined, it default to the value of Target Content Code.
    • Add URL - If users can add new items, this defines the URL of the add page. If not defined, it defaults to the value of Target URL.
  • Allow deleting - Enable to allow users to delete from the grid.

Notes

  • While you can add this to a content record as a standalone iPart, it is not recommended as it is included in other iParts Finder/Adder functionality.

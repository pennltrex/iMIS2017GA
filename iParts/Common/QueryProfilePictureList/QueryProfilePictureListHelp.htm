Query Profile Picture List

  • Name - Enter a name for the content item. This text only appears during design mode (configuration).
  • Title - Enter the text to display above this iPart on the rendered page. If blank, no title displays.
  • Do not render in design mode - Enable this option to prevent content record data from being loaded in the content editor. This improves performance during design time (configuration).
  • Display a border around this content - Enable this option to display a border around this iPart on the rendered page.
  • List source query - Click select and browse to the IQA query that you want to see results for in the list.
  • Max. returned - Enter the total number of records to be displayed.
  • Data column - Select a data column if you want to dynamically filter the query results based on the value of that particular property. If the Data column and/or Query string parameter are not defined, then the results will not be filtered beyond what is defined within the source query itself.  Note: Each property you select on the Display tab of your query in IQA is available as a Data column option.
  • Query string parameter - If dynamically filtering the query based on the value of a Data column, enter the paramater name by which you want to filter values.
  • Description - If you want a description to be displayed below the title and above the list, enter the text for the description here. The variable {0} can be used in the description to display the total record count number. For example, "{0} people also bought this product".
  • No records found message - Message displayed if no records are returned. If this field is blank, no message is displayed to the user and the iPart will not be shown on the page.
  • Click Url - The url used when the image is clicked. The url will append the ID value of the record. If blank the image is not clickable. The url should be relative to the website root; for example the link to http://www.yoursite.com/Member/iMIS/ContactManagement/ContactLayouts/Account_Page.aspx should be entered as Member/iMIS/ContactManagement/ContactLayouts/Account_Page.aspx.
  • Image dimension - The dimension of the thumbnail. (Small, Medium or Large)
  • Display orientation - The display orientation of the list. (Horizontal or Vertical)

Notes

  • For a sample query that works with this iPart, see $/Samples/QueryProfilePictureList in IQA.
  • You must set one of your display column's aliases to key_Id in order for this iPart to display properly within a content record.
  • For details, see "Displaying query results in iParts".

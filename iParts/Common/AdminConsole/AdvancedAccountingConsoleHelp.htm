Advanced Accounting Console

  • Name - Enter a name for the content item. This text appears during design mode (configuration).
  • Title - Enter the text to display above this iPart on the rendered page. If blank, no title displays.
  • Do not render in design mode - Select this option to prevent content record data from being loaded in the content editor. This improves performance during design time (configuration).
  • iPart CSS class - (optional) Enter the name of a CSS class to associate with this iPart, which will add a DIV so named around the iPart. Add this CSS class to an appropriate style sheet. Using such classes lets you apply special formatting to targeted iParts in a reusable way.
  • Display a border around this content - Select this option to display a border around this iPart on the rendered page.
  • Display content within a collapsible panel - Enable to allow users to minimize the panel within the page, to work with dense pages more efficiently.
    • Show the content as collapsed - Enable to display the panel minimized by default.
      Tip: Be sure that the Title makes clear what data is hidden from view.
  • Display on screen sizes (for responsive themes only):
    • Extra-small screens - When disabled, this content item is hidden on extra small devices, such as smartphones, where the width of the screen is less than 768px.
    • Small screens - When disabled, this content item is hidden on small devices, such as tablets, where the width of the screen is greater than 768px and less than 991px.
    • Medium screens - When disabled, this content item is hidden on medium devices, such as laptops, where the width of the screen is greater than 992px and less than 1199px.
    • Large screens - When disabled, this content item is hidden on large devices, such as wide computer monitors, where the width of the screen is greater than or equal to 1200px wide.
      Note: When Easy Edit is enabled, all content items for a page are displayed, regardless of the Display on screen sizes (for responsive themes only) options that are selected.

RDP Connection

  • Server address - The address of the terminal server on which the Advanced Accounting Console is installed.
  • Windows domain - The domain name to use when logging onto the terminal server.
  • Windows username - The username for logging onto the terminal server.
  • Windows password - The password for logging onto the terminal server.
  • Width - The width in pixels to display the remote desktop screen.
  • Height - The height in pixels to display the remote desktop screen.

Advanced Accounting Console

  • Command line - The command line to launch the Advanced Accounting Console on the terminal server. This will be the path of the Omnis7.exe. You must include imis4.lbr as the first command line parameter.
  • Working directory - The working directory. This will be the parent directory of the path specified in Command Line. This directory contains all the .lbr files.

Notes

This iPart should be placed on its own on a content page. It will take full control of the content page and redirect to a specific page which hosts the RDP control. No other iParts will be displayed. Although not essential it is generally advisable to configure a navigation link to open the content page in a new window.

Only Full and Casual users will be able to access to use this facility, all other users will see "Access denied".

 The page hosting the RDP control will contain the connection details for the terminal server. Some consideration should therefore be given to security. The page will use the same transport protocol as the content page containing the iPart i.e. if the content page is opened using https then the page hosting the control will also use https. Public and Guest users will never have access to the connection details.

The terminal server must be specifically configured to allow the Advanced Accounting Console to be remotely launched.

The Advanced Accounting Console installed on the terminal server must be preconfigured to connect to the same iMIS application server as the website. This connection information is not passed through from the website.
